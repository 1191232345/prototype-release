# PRD - 价卡管理

> 评审对照文档：对照原型时以 **第 3 章「页面交互规格」** 为准。

---

## 1. Executive Summary

- **Problem Statement**：价卡管理系统缺乏标准化文档和原型对照机制，费用项配置分散难以统一管理，评审效率低
- **Proposed Solution**：将价卡作为"套餐"概念，引用现有费用规则并配置折扣，实现费用项的打包管理
- **Business Model**：价卡是费用项的套餐，将多个费用规则打包为一个整体，为客户提供统一的折扣方案
- **Success Criteria**：
  - 价卡支持引用快递费、入库费、增值服务等多种费用规则
  - 每个费用项可独立配置折扣方式（折扣率、增减、一口价）
  - 价卡列表清晰展示包含的费用项数量和类型
  - 所有页面有规范的 PRD 文档和原型对照

---

## 2. User Experience & Functionality

### User Personas

- **运营人员**：负责价卡的创建、编辑、费用配置，使用列表页筛选查询，使用创建/编辑页维护价卡
- **财务人员**：负责价卡的审核和费用核算，使用详情页查看完整信息

### User Stories

- **Story 1**：作为运营人员，我希望快速筛选价卡，以便找到需要维护的价卡配置
- **Story 2**：作为运营人员，我希望通过弹窗快速创建价卡，以便减少页面跳转提高效率
- **Story 3**：作为运营人员，我希望在独立页面维护费用项，以便专注管理特定费用组

### Acceptance Criteria

- [ ] 价卡列表页支持按名称、创建人、更新人、时间筛选
- [ ] 创建价卡弹窗自动填写默认名称
- [ ] 编辑价卡仅修改基本信息，费用项通过维护页面管理
- [ ] 费用维护页面支持添加、编辑、删除费用项
- [ ] 折扣配置支持标准折扣和阶梯折扣

### Non-Goals

- 真实后端 API 对接
- 复杂权限控制
- 其他业务系统集成

### 按钮交互逻辑说明

**新增价卡按钮**：
- 点击后立即跳转到创建价卡页面
- 页面初始化时自动填写价卡名称为"新价卡XXX"（XXX为自动递增编号）
- 用户可修改名称和描述，填写完成后点击"保存"创建价卡
- 保存成功后自动跳转回列表页，新创建的价卡显示在列表顶部

**查询按钮**：
- 点击后根据当前所有筛选条件执行过滤
- 支持多条件组合筛选（AND逻辑）
- 筛选结果实时更新到表格中
- 空结果时显示空状态提示："暂无符合筛选条件的价卡数据"

**重置按钮**：
- 点击后清空所有筛选字段，恢复到初始状态
- 不会清空表格数据，只清空筛选条件
- 重置后自动执行一次全量查询，显示所有价卡

### 快递费用项交互逻辑

**级联选择逻辑**：
- 选择渠道 → 过滤承运商下拉选项（只显示该渠道的承运商）
- 选择承运商 → 过滤费用类型下拉选项（只显示该承运商的费用类型）
- 选择费用类型 → 过滤费用名称下拉选项（只显示该费用类型的费用项）
- 选择费用名称 → 自动带出计费单位、单价（从快递费规则数据加载）

**折扣计算逻辑**：
- 无折扣：预计金额 = 单价
- 折扣率：预计金额 = 单价 × (1 - 折扣值/100)
- 增减：预计金额 = 单价 - 折扣值（不能为负数）
- 一口价：预计金额 = 折扣值（直接替换单价）

**表格操作**：
- 新增行：点击"新增"按钮，添加一行空数据
- 删除行：点击"删除"按钮，删除当前行
- 复制行：点击"复制"按钮，复制当前行数据到新行
