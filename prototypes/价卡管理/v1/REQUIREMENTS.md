# 需求文档 - 价卡管理

## 现状

价卡管理系统目前是完整的 HTML/JS 实现，包含价卡列表、创建、编辑、详情查看、费用维护等功能。费用维护页面分散为三个独立页面（库内费用、快递费用、其他费用），缺乏统一的文档和原型对照机制，难以进行规范化的评审和沟通。

## 痛点

- 费用维护页面分散，难以统一管理
- 缺乏规范的文档和对照机制，评审效率低
- 代码逻辑复杂，交互流程不够清晰
- 缺乏标准化的 JSON Spec 格式，不利于后续迭代和维护

## 诉求与目标

将现有价卡管理系统迁移到 prototype-archive 原型系统，实现：
- 标准化的 JSON Spec 格式定义页面规格
- 规范的 PRD 文档和原型对照，便于评审和沟通
- 保持现有功能和交互逻辑不变
- 为后续迭代和维护提供基础

## 范围边界

### 做

- 价卡列表页：筛选、表格、行操作（查看/编辑/删除/维护费用）
- 创建价卡页：填写基本信息、配置费用项、折扣配置
- 编辑价卡页：修改基本信息
- 费用维护页：三个独立页面（库内费用、快递费用、其他费用）
- 折扣配置功能：标准折扣、阶梯折扣
- 数据源引用：入库费规则、增值服务规则等

### 不做

- 真实后端 API 对接
- 复杂权限控制
- 其他业务系统集成

## 验收标准

- [ ] 价卡列表页支持按名称、创建人、更新人、时间筛选
- [ ] 列表页行操作包含查看（弹窗）、编辑（弹窗）、删除（确认）、维护费用（跳转）
- [ ] 创建页包含基本信息填写、费用项配置（Tab切换）、折扣配置
- [ ] 编辑页仅修改基本信息，费用项通过维护页面管理
- [ ] 费用维护页支持添加、编辑、删除费用项，配置折扣
- [ ] 折扣配置支持标准折扣（折扣率、增减、一口价）和阶梯折扣
- [ ] 所有页面有逻辑说明文档
