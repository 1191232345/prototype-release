# 需求文档 - SKU 信息填充

## 现状

SKU 的尺寸、重量、图片由仓库线下提供，客服人员手工录入并上传至系统，与 SKU 基础数据维护耦合在同一人工流程中。

## 痛点

- 客服人工处理量大，重复录入多
- 手工上传易出错，尺寸/重量/图片与 SKU 对应关系缺乏系统化约束

## 诉求与目标

建立 SKU 信息补充工单机制，将「客服发起 → 仓库补充 → 客服审核」流程线上化：

- 客服创建工单（选 SKU、填备注/优先级/仓库代码），驱动仓库补充
- 仓库通过系统上传尺寸、重量、图片（仓库端不在本次原型范围）
- 客服审核已提交的 SKU 信息，可编辑后完结（写回 SKU 基础数据）或退回
- 已完结工单只读，不可再改

## 范围边界

### 做

- 客服端：工单列表、新建工单、工单详情页、工单审核页
- PDA 端：任务执行页（仓库人员提交 SKU 尺寸/重量/图片）
- 工单状态：待仓库处理 / 待客服审核 / 已完结 / 已退回
- 工单维度：客户代码、备注、优先级、仓库代码、关联 1 个 SKU（单工单单 SKU）
- SKU 选择：新建/编辑弹窗内单选 SKU，展示尺寸、重量
- 列表支持 Excel 批量导入新建工单、导出当前筛选结果（含 SKU 尺寸/重量/图片）
- SKU 补充字段：尺寸、重量、图片
- 审核时仅展示/处理已提交的 SKU
- 完结后写回 SKU 基础数据；已完结只读

### 不做

- 仓库端完整管理系统（仅 PDA 端）
- 与外部 WMS/基础数据的实时对接

## 验收标准

- [ ] 列表可按工单号、客户代码、状态、仓库、优先级、创建人、创建时间筛选
- [ ] 列表支持导入（批量新建工单）与导出（当前筛选结果，含 SKU 尺寸/重量/图片）
- [ ] 每个工单仅绑定 1 个 SKU，SKU 进度展示 0/1 或 1/1
- [ ] 各状态行操作正确：待仓库处理可查看/编辑；待客服审核可审核；已退回可编辑；已完结只读
- [ ] 新建/编辑工单：客户 + 单 SKU + 备注/优先级/仓库，提交后进入待仓库处理
- [ ] 查看跳转详情页（左右布局 + 操作时间轴）；审核跳转审核页
- [ ] 审核可退回（必填原因）或完结（写回基础数据确认）
- [ ] 退回后仓库重传或客服修改，工单回到待客服审核
- [ ] PDA 端可查看工单基本信息、填写绑定 SKU 的尺寸/重量/上传图片
- [ ] PDA 端支持暂存和提交操作
- [ ] PDA 端提交后工单变为待客服审核状态
