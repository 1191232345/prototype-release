# 变更记录 - SKU 信息填充 v1

## 2026-07-08 · 列表新增 SKU 筛选项

- **筛选项新增**：列表 filters 新增「SKU」文本输入框，支持 SKU 编码模糊匹配（仅匹配编码，不含名称）
- **PRD 同步**：`{#list.filters}` 表格新增 SKU 行（输入方式：文本；匹配逻辑：模糊匹配；触发时机：点击查询；数据来源：本页列表）

## 2026-07-07 · 导入导出 + 单工单单 SKU

- **导入**：列表「导入」打开批量导入弹窗，Excel 模板字段为「客户代码 + 仓库代码 + SKU 编码 + 优先级 + 备注」，工单号系统生成，仅批量新建
- **导出**：列表「导出」按当前筛选结果导出全部列，含 SKU 尺寸/重量/图片
- **单 SKU 约束**：新建/编辑弹窗每工单仅选 1 个 SKU；列表 SKU 进度改为 0/1 或 1/1
- **删除订单号**：移除出库单关联字段及自动填充
- **Pattern 扩展**：`SkuFillList` 接入 importModal/exportList；`SkuFillFormSkuTable` 支持 `maxRows: 1`
- **PRD 同步**：新增 `{#list.import-modal}`，更新 `{#list.main-buttons}` `{#list.form-modal.*}` `{#list.init}` PDA 相关章节

## 2026-07-07 · 新增 PDA 端任务执行页

- **PDA 端新增**：新增 PDA 端任务执行页（pattern: `sku-fill-pda-task`），用于仓库人员执行 SKU 信息补充任务
  - 工单基本信息展示（工单号、客户代码、仓库代码、优先级、备注、退回原因）
  - SKU 信息补充表单（尺寸/重量/图片上传）
  - 暂存/提交按钮，提交后工单变为待客服审核
- **Schema 扩展**：新增 `sku-fill-pda-task` pattern、`pageMode: "pda"`、`layout: "pda-sku-form"`
- **PRD 同步**：新增第 3.5 章 PDA 端交互规格（含锚点 `{#pda.init}` `{#pda.info}` `{#pda.sku-form}` `{#pda.validation}` `{#pda.main-buttons}`）
- **范围更新**：REQUIREMENTS.md 将 PDA 端从"不做"移至"做"，新增验收标准
- **Mock 数据**：新增 `pages/pda.details.json`（wo-001、wo-003 两条样例）
