# PRD - SKU 信息填充

> 评审对照：第 3 章「页面交互规格」与原型逐项核对。
>
> **产品语言约定**：使用中文业务名称；不写字段 ID、key、表名、mock 变量；数据来源用菜单路径描述。

---

## 1. Executive Summary

- **Problem Statement**：SKU 尺寸、重量、图片由仓库线下提供、客服手工上传，工作量大且易出错。
- **Proposed Solution**：建立 SKU 信息补充工单流程，客服发起 → 仓库补充 → 客服审核完结，尺寸/重量/图片写回 SKU 基础数据。
- **Success Criteria**：
  - 客服可在 3 步内完成工单创建（选 SKU → 填信息 → 提交）
  - 支持 Excel 批量导入新建工单、按筛选结果导出（含 SKU 尺寸/重量/图片）
  - 每个工单仅绑定 1 个 SKU，SKU 进度以 0/1 或 1/1 展示
  - 各工单状态行操作准确率 100%
  - 审核页可编辑已提交 SKU、退回（必填原因）、完结（写回确认）

## 2. User Experience & Functionality

### User Personas

- **客服人员**：创建补充工单、审核仓库提交的 SKU 信息、退回或完结工单
- **仓库人员**：使用 PDA 端接收任务并提交 SKU 尺寸/重量/图片

### User Stories

- **Story 1**：作为客服，我希望按状态/仓库筛选工单，以便快速定位待审核任务。
- **Story 2**：作为客服，我希望批量导入工单或导出筛选结果，以便提升批量处理效率。
- **Story 3**：作为客服，我希望退回时填写原因，以便仓库明确重传要求。

### Acceptance Criteria

- [ ] 列表支持工单号、客户代码、状态、仓库、优先级、创建人、创建时间筛选
- [ ] 列表支持导入（批量新建工单）与导出（当前筛选结果，含 SKU 尺寸/重量/图片）
- [ ] 每个工单仅绑定 1 个 SKU，SKU 进度展示为 0/1 或 1/1
- [ ] 行操作按状态动态显示（查看/审核/编辑）
- [ ] 新建打开弹窗；查看跳转详情页；审核跳转审核页（左右布局 + 操作时间轴）
- [ ] 审核页支持编辑已提交 SKU（含图片）、退回（必填原因）、完结（写回确认）
- [ ] 已完结工单只读

### Non-Goals

- 仓库端完整管理系统（仅 PDA 端）
- 真实后端 API 对接
