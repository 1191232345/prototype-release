# PRD - SKU 信息填充 PDA 端

> 评审对照文档：对照原型时以 **第 3 章「页面交互规格」** 为准。

---

## 1. Executive Summary

- **Problem Statement**：仓库人员缺少移动端界面，无法高效执行 SKU 信息补充任务
- **Proposed Solution**：创建独立 PDA 端项目，提供任务列表 + 任务执行两级页面，支持尺寸/重量/图片填写和提交
- **Success Criteria**：
  - 仓库人员可在 5 分钟内完成单个 SKU 信息补充
  - 暂存和提交功能正常，数据提交后工单状态自动更新

---

## 2. User Experience & Functionality

### User Personas

- **仓库人员**：使用 PDA 设备接收任务并提交 SKU 尺寸/重量/图片

### User Stories

- **Story 1**：作为仓库人员，我希望查看待处理任务列表，以便选择需要执行的工单。
- **Story 2**：作为仓库人员，我希望快速查看工单基本信息，以便了解任务背景。
- **Story 3**：作为仓库人员，我希望填写 SKU 尺寸/重量并上传图片，以便完成信息补充。

### Acceptance Criteria

- [ ] PDA 端可浏览任务列表，点击进入任务执行页
- [ ] PDA 端可查看工单基本信息、填写 SKU 尺寸/重量/上传图片
- [ ] PDA 端支持暂存（部分 SKU）和提交（全部 SKU）
- [ ] PDA 端提交后工单变为待客服审核状态
- [ ] 移动端布局适配，表单控件友好易用

### Non-Goals

- 扫码进入功能
- 真实后端 API 对接

## 4. 实现约束（可选）

- **关联功能**：商品管理 > SKU 列表（SKU 编码/名称）、SKU 信息填充 > SKU 信息补充工单（工单信息）
- **权限说明**：仓库人员可查看工单信息、填写 SKU 信息、暂存/提交
- **性能预期**：PDA 端加载时间 ≤ 2 秒，图片上传响应时间 ≤ 3 秒

---

## 5. Risks & Roadmap

### Phased Rollout

- **MVP**：PDA 端任务列表 + 任务执行页（本期）
- **v1.1**：扫码进入功能
- **v2.0**：实时 API 对接 + 状态同步

### Technical Risks

- 移动端图片上传性能需优化
- 部分 SKU 提交时状态判定需与后端对齐
