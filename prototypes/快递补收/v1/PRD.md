# PRD - 快递补收

> 评审对照：第 3 章「页面交互规格」与原型逐项核对。

---

## 1. Executive Summary

- **Problem Statement**：海外仓尾程运输中，客户下单尺寸与承运商实测尺寸不一致，目前依赖线下账单与人工核算差价，工作量大且易出错。
- **Proposed Solution**：提供快递补收管理界面，支持 Excel 导入承运商实测数据、按运单号自动匹配订单并算价、运营确认补差记录。
- **Success Criteria**：
  - 运营可在 3 步内完成一批导入（选价卡 → 上传 Excel → 开始导入）
  - 导入后系统自动计算差价，列表即时展示匹配结果与金额
  - 待确认/已确认/匹配失败三种状态展示准确率 100%

---

## 2. User Experience & Functionality

### User Personas

- **运营人员**：导入承运商账单、核对差价、确认或删除补收记录
- **产品经理**：查看详情、验收原型交互与算价展示逻辑

### User Stories

- **Story 1**：作为运营，我希望批量导入承运商 Excel，以便替代线下逐单核算。
- **Story 2**：作为运营，我希望系统自动按运单号匹配订单并计算差价，以便快速发现需补收或退差的运单。
- **Story 3**：作为运营，我希望核对无误后一键确认，以便标记处理完成。

### Acceptance Criteria

- [ ] 列表支持运单号、客户、状态、匹配结果、导入人、配送渠道、导入时间（起止）筛选
- [ ] 导入通过列表弹窗完成：选文件识别条数 → 点击「确认导入」进入下一步；≤1000 条配置价卡后算价；>1000 条提交异步任务
- [ ] 列表支持行勾选与全选；批量确认/删除支持勾选范围或全量（未勾选时），均须二次确认
- [ ] 列表展示配送渠道列，与订单匹配结果一致
- [ ] 匹配成功且待确认的行可确认，匹配失败不可确认
- [ ] 查看打开详情弹窗，展示尺寸对比与费用对比
- [ ] 删除、批量删除、确认、批量确认操作有二次确认

### Non-Goals

- 1 对多 / 多对多运单匹配
- 运营编辑导入结果或重算
- 确认后生成补收单、推送客户
- 客户侧确认（后续迭代）

## 4. Technical Specifications

- **Architecture Overview**：Flow 原型（list 单页 + 导入弹窗），Pattern 驱动渲染，Spec JSON 按类型拆分
- **Integration Points**：原型阶段使用 mock 数据；正式版对接订单系统（运单号匹配）、价卡规则引擎（算价）
- **Security & Privacy**：承运商账单含运单与金额信息，需运营权限访问

---

## 5. Risks & Roadmap

### Phased Rollout

- **MVP（v1）**：Excel 导入 + 自动匹配算价 + 运营确认 + 详情查看
- **v1.1**：下载模板、导入结果预览、确认二次弹窗文案优化
- **v2.0**：客户确认、生成补收单、对接计费系统；1 对多匹配

### Technical Risks

- 运单号匹配失败率需监控，需明确失败数据处理流程
- 价卡规则变更可能导致历史记录算价口径不一致
