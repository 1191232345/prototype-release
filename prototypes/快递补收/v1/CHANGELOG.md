# 变更记录 - 快递补收 v1

## 2026-07-05
- **状态文案自动推导**
  - 渲染层从 `filters.status.options` 自动推导列表/详情状态标签
  - 移除冗余 `statusLabels` 配置，筛选项为唯一文案来源
  - 同步更新 PRD.md `{#list.columns}`

## 2026-07-05
- **状态列展示统一**
  - 新增 `statusLabels` 配置，列表与详情弹窗状态标签统一为「待确认」「已确认」
  - 修复渲染层 Badge 默认「草稿/已发布」与 不一致的问题
  - 同步更新 PRD.md `{#list.columns}`

## 2026-07-05
- **批量操作全量兼容**
  - 未勾选时，批量确认/删除作用于当前列表全部记录
  - 有勾选时按选中范围；按钮显示 `(N)` 或 `(全量)`
  - 二次确认弹窗文案区分勾选与全量两种场景

## 2026-07-05
- **批量确认**
  - 工具栏新增「批量确认」，支持勾选与表头全选
  - 批量确认、批量删除均弹出二次确认对话框
  - 批量确认仅处理匹配成功且待确认的记录
  - 同步更新 PRD.md `{#list.main-buttons}`、`{#list.state-flow}`

## 2026-07-05
- **检索与列表列调整**
  - 检索新增：配送渠道（下拉）、导入时间起止（datetime）
  - 列表移除「承运商金额」列（详情弹窗费用对比仍保留）
  - 同步更新 PRD.md `{#list.filters}`、`{#list.columns}`

## 2026-07-05
- **导入弹窗优化：异步阈值**
  - 移除「解析文件」步骤，选文件后自动识别条数
  - 新增 `asyncThreshold: 1000`；超过阈值直接提交异步任务
  - 异步路径 Toast「已添加到异步任务，请稍后在任务中心查看」
  - Spec 新增 `filePresets` 模拟小文件（25条）与大文件（1500条）便于验收
  - 同步更新 PRD.md `{#list.import-modal}`

## 2026-07-05
- **列表交互优化**
  - 「导入」改为列表页弹窗（`importModal`），不再跳转独立表单页；`flow.json` 移除 form 页
  - 新增批量删除：表格支持行勾选 + 表头全选，工具栏「批量删除」二次确认
  - 主表新增「配送渠道」列（`deliveryChannel`），详情弹窗同步展示
  - 同步更新 PRD.md：`{#list.main-buttons}`、`{#list.columns}`、`{#list.import-modal}` 等锚点章节

## 2026-07-05
- **导入页两步流程优化**
  - 新增专用 Pattern `express-supplement-import-form`
  - 步骤 1：上传 Excel → 点击「解析文件」提取客户代码
  - 步骤 2：解析摘要 + 客户价卡配置表（每客户一行，可切换价卡）
  - 用户确认价卡无误后，点击「确认并计算差价」才执行匹配算价
  - 同步更新 PRD.md 表单页交互规格

## 2026-07-05（初版）
- **v1 初版创建**
  - 列表页：运单号/客户/状态/匹配结果/导入人筛选，展示补收记录表格
  - 导入表单页：价卡模式（自动匹配/手动选择）+ Excel 上传
  - 详情弹窗：基本信息、尺寸对比、费用对比、导入与确认信息
  - Mock 数据：待确认（正差价）、已确认（负差价）、匹配失败各一条
  - 行操作：查看、确认（publish）、删除；匹配失败仅查看/删除
