# 需求文档 - 快递补收

## 现状

海外仓尾程场景下，客户下单时的包裹尺寸与承运商实测尺寸经常不一致。承运商目前以线下账单提供实测数据（运单号、长宽高/重量、金额等），运营人工核对、按规则核算「真实价格」与「差价」，工作量大且易出错。

## 痛点

1. **人工处理量大**：逐单核对尺寸与金额，重复劳动多。
2. **出错概率高**：手工计算与核对缺少系统约束，易产生差错。

## 诉求与目标

1. **线上导入**：支持运营导入承运商 Excel（含运单号、实测长宽高/重量、承运商金额）。
2. **系统自动算价**：按运单号匹配订单，基于现有已维护的计价规则计算真实价格与差价；**先解析 Excel 中的客户代码，逐客户配置价卡，确认后再执行算价**。
3. **无论是否有补收金额，均需计算并展示差价**。
4. **运营确认**：现阶段由运营确认，确认后仅标记为「已确认」；后续再开放客户确认。

## 范围边界

### 做

- Excel 导入承运商实测数据（两步：解析客户代码 → 逐客户配置价卡 → 确认后算价）
- 按运单号 1:1 自动匹配客户订单
- 按现有规则/价卡自动计算真实价格、原价与差价
- 列表展示导入记录（一条 = Excel 一行）
- 详情查看（尺寸对比、价格与差价等）
- 运营「确认」（待确认 → 已确认）
- 待确认、已确认状态下均可「删除」

### 不做

- 1 对多 / 多对 1 / 多对多匹配
- 运营对导入结果的编辑（含改尺寸、改价卡、重算）
- 确认后生成补收单、推送客户、对接计费等下游动作
- 客户侧确认（后续迭代）

## 验收标准

- [ ] 导入表单先解析客户代码并列举，每个客户单独配置价卡，确认后再计算差价
- [ ] 按运单号自动匹配订单并计算真实价格与差价
- [ ] 列表可筛选运单号、客户、状态、匹配结果、导入人、配送渠道、导入时间（起止）
- [ ] 导入通过列表弹窗完成：选文件 → 确认导入 → ≤1000 条配置价卡后算价；>1000 条提交异步任务
- [ ] 列表支持勾选与全选，可批量删除（二次确认）
- [ ] 列表展示配送渠道列
- [ ] 匹配成功且待确认的记录可确认；匹配失败不可确认
- [ ] 待确认、已确认状态均可删除，删除有二次确认
- [ ] 详情弹窗展示尺寸对比、费用对比及导入/确认信息

## 变更记录

### 2026-07-05 · 状态文案自动推导

- 列表/详情状态标签从 `filters.status.options` 自动推导，无需单独维护 `statusLabels`
- 改筛选项文案即同步列表展示，避免与 PRD 脱节

### 2026-07-05 · 状态列展示统一

- 列表状态列与详情弹窗统一展示「待确认」「已确认」（非规则配置的「草稿/已发布」）
- Spec 新增 `stLabels` 驱动展示文案，与 PRD 对齐

### 2026-07-05 · 批量操作全量兼容

- 批量确认/删除：有勾选按选中行，无勾选按当前列表全量
- 按钮标注 (N) 或 (全量)，二次确认弹窗区分两种范围

### 2026-07-05 · 批量确认

- 新增批量确认按钮，勾选/全选后二次确认
- 批量确认与批量删除均须二次确认

### 2026-07-05 · 检索与列表列调整

- 检索新增配送渠道、导入时间（起止）
- 列表移除承运商金额列

### 2026-07-05 · 导入弹窗异步阈值

- 移除解析步骤，选文件自动识别条数
- 超过 1000 条走异步任务并提示用户

### 2026-07-05 · 列表交互优化

- 导入改为列表页弹窗，支持勾选与全选批量删除
- 主表新增配送渠道列

### 2026-07-05 · 导入页两步流程优化

导入表单改为：上传解析 → 列举客户代码并逐客户配置价卡 → 确认后计算差价。

### 2026-07-05 · v1 初版创建

创建快递补收 Flow 原型：列表 + 导入表单 + 详情弹窗，支持承运商 Excel 导入、自动算价补差与运营确认。
