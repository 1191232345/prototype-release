# PRD - 快递费管理

> 评审对照：第 3 章「页面交互规格」与原型逐项核对。

---

## 1. Executive Summary

- **Problem Statement**：海外仓快递费公报价分散在 Excel 中，无法归档溯源，每次更新需手动整理，效率低且易出错。
- **Proposed Solution**：建立快递费管理系统，支持列表检索、表单维护基础价分区报价与附加费配置，实现新增/编辑/复制/导入导出。
- **Success Criteria**：
  - 产品人员可在 5 步内完成一条基础价或附加费配置（筛选定位 → 新增 → 填基础信息 → 维护报价/附加费 → 保存）
  - 分区报价重量段「起/止」联动准确率 100%
  - 编辑模式下费用类型不可变更，避免业务歧义
  - 费用类型切换后，下方配置区域自动联动切换

## 2. User Experience & Functionality

### User Personas

- **产品管理人员**：维护海外仓快递费公报价，负责基础价、附加费的新增、调整、归档
- **产品经理**：验收原型交互，对照 PRD 核对字段与按钮行为

### User Stories

- **Story 1**：作为产品管理人员，我希望按渠道/承运商/费用类型筛选，以便快速定位目标报价。
- **Story 2**：作为产品管理人员，我希望在表单页统一编辑基础信息与配置区域，以便一次保存完整配置。
- **Story 3**：作为产品管理人员，我希望查看详情时只读浏览基本信息，复杂报价在表单页维护。
- **Story 4**：作为产品管理人员，我希望选择「附加费」或「其他费用」时，表单下方展示附加费管理区域，以便维护属性费用项、单价、分区等信息。

### Acceptance Criteria

- [ ] 列表可按渠道、承运商、费用类型、创建人筛选
- [ ] 列表第一列为名称；行操作支持查看、编辑、复制、删除
- [ ] 列表包含基础价、附加费、其他费用三种费用类型的 mock 数据
- [ ] 新增时表单字段与配置区域均为空；编辑时自动预填充
- [ ] 编辑模式下费用类型禁用；新增模式下可选
- [ ] 选择「基础价」时，下方展示分区报价配置
- [ ] 选择「附加费」时，下方展示附加费管理区域（费用项：燃油附加费等）
- [ ] 选择「其他费用」时，下方展示其他费用管理区域（费用项：签名服务、保险服务）
- [ ] 分区报价「起」继承上一行「止」且不可编辑；「止」可编辑并联动下一行「起」
- [ ] 支持添加/删除重量段、导入分区报价、单价实时编辑
- [ ] 附加费管理区域支持添加行、删除行、导入配置
- [ ] 附加费管理区域分区支持下拉多选

### Non-Goals

- 草稿/生效/失效等状态管理
- 真实后端 API 对接（原型阶段 mock）

## 5. Risks & Roadmap

### Phased Rollout

- **MVP（当前原型）**：列表 + 表单 + 分区报价表格编辑 + 附加费管理 + 详情弹窗（仅基本信息）
- **v1.1**：复制时预填源行数据、导入/导出真实逻辑、editData 含配置区域数据
- **v2.0**：对接真实 API、批量导入列表、归档版本管理

### Technical Risks
