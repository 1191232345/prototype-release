# PRD - 规则配置

> 评审对照：第 3 章「页面交互规格」与原型逐项核对。

---

## 1. Executive Summary

- **Problem Statement**：业务规则分散在 Excel 与口头约定中，运营缺乏统一配置入口，查找慢且易误操作。
- **Proposed Solution**：提供规则配置管理界面，支持列表检索、详情查看、表单新建/编辑，状态流转清晰。
- **Success Criteria**：
  - 列表筛选响应时间 < 500ms（前端 mock）
  - 运营可在 3 步内完成新建规则配置（选客户 → 填表单 → 发布）
  - 草稿/已发布/已作废状态区分准确率 100%

---

## 2. User Experience & Functionality

### User Personas

- **运营人员**：日常创建、发布、作废规则配置
- **产品经理**：查看配置详情、验收原型交互

### User Stories

- **Story 1**：作为运营，我希望在左侧选择客户、右侧查看各有效期周期的价卡来源，以便快速确认「谁在什么时间用什么价卡」。
- **Story 2**：作为运营，我希望草稿可编辑/发布/删除，已发布可作废，以便状态流转可控。
- **Story 3**：作为运营，我希望查看详情时不离开列表页，以便对比多条配置。

### Acceptance Criteria

- [ ] 列表支持客户、生效状态、创建人、创建时间、更新人、更新时间筛选
- [ ] 列表以左右分栏展示：左侧客户列表，右侧该客户的有效期规则列表
- [ ] 行操作仅对二级（有效期）行生效；草稿/已发布/已作废状态区分准确
- [ ] 新增/编辑跳转表单页，查看打开详情弹窗
- [ ] 删除、复制操作有二次确认
- [ ] 表单采用左右分栏：左侧客户授权仓库列表，右侧按仓配置费用；仓库不可手动增删
- [ ] 费用区支持「快速填入」：价卡模板带入与复用其他仓规则二选一
- [ ] 费用导航：大类分段控件（快递费 / 库内费用 / 其他费用）+ 子项 Tab，同一时刻仅展示一个大类
- [ ] 费用项表格支持增删行；可通过价卡模板或复用仓库带入初始规则
- [ ] 同客户下 published 规则生效期不可重叠；发布前展示影响预览
- [ ] 列表展示生效状态（生效中 / 待生效 / 已过期 / 未发布）；发布时冻结规则快照

### Non-Goals

- 真实后端 API 对接
- 复杂审批流与权限体系
- 追溯改价重算任务（单独流程，本期不做）

## 3.1 列表页 · 生效逻辑说明 {#list.effective-logic}

### 计费匹配规则（引擎侧）

- 匹配键：**客户**（同客户维度）
- 条件：**已发布** 且 `startTime ≤ event_time ≤ endTime`（**闭区间，含起止**）
- 失效时间默认取当日 **23:59:59**；新规则默认从 **上一规则结束日 +1 天的 00:00:00** 开始
- 同一客户在任意时刻最多 **1 条** published 规则生效
- 每条计费明细落库 `rule_version_id`，关联发布时冻结的快照

### 生效状态计算（列表展示）

| 生效状态 | 计算条件 | 展示样式 |
|----------|----------|----------|
| 未发布 | 记录状态为草稿 | 灰色 |
| 待生效 | 已发布且当前时间早于开始时间 | 蓝色 |
| 生效中 | 已发布且当前时间在生效周期内 | 绿色 |
| 已过期 | 已发布且当前时间晚于结束时间，或记录已作废 | 灰色/删除线 |

### 发布逻辑说明 {#list.publish-logic}

> 触发入口：列表行「发布」、表单页「发布」。用户确认后执行，暂存不触发。

**核心原则**：发布 = **生成新规则** +（如有重叠）**修改旧规则失效时间**。永久替代，不换回旧价。

| 步骤 | 系统动作 | 数据变更 |
|------|----------|----------|
| 1. 预览 | 计算生效判定；扫描同客户+费用类型的时间重叠 | 无写入 |
| 2. 生成新规则 | 草稿 → 已发布；冻结价卡+费用项快照 | **新增**一条规则，周期 = 表单 [start, end] |
| 3. 处理重叠 | 旧规则 `effective_end_time` 截断为「新 start 前一日 23:59:59」 | **修改**旧规则失效时间 |
| 4. 生效刷新 | 开始日期 ≤ 今天 → 新规则立即生效；> 今天 → 待生效 | 列表生效状态动态更新 |

| 场景 | 旧规则 | 新规则 |
|------|--------|--------|
| 无时间重叠 | 不修改 | 生成新规则，按周期自然待生效/生效 |
| 有重叠 · 开始 ≤ 今天 | 截断 end，立即停用 | 生成新规则，立即生效 |
| 有重叠 · 开始 > 今天 | 截断 end，新 start 前继续执行 | 生成新规则，待生效 |

## 3.1 列表页 · 状态变更流程 {#list.state-flow}

| 当前状态 | 允许操作 | 目标状态 | 说明 |
|----------|----------|----------|------|
| — | 新建 / 暂存 | 草稿 | 表单页暂存 |
| — | 新建 / 直接发布 | 已发布 | 可跳过草稿 |
| 草稿 | 发布 | 已发布 | 列表行操作「发布」 |
| 已发布 | 作废 | 已作废 | 不可逆 |
| 已发布 | 作废 | 已作废 | 不可逆；历史计费仍追溯快照 |
| 已作废 | 复制 | 新草稿 | 复制新增，名称 +「（副本）」 |
| 已发布 | 变更内容 | — | **禁止原地编辑**；须新建规则并指定新生效期 |

> 已发布规则内容不可修改；任何变更通过「新建/复制 → 发布新版本」完成，保证计费可追溯。

## 3.2 表单页 · 发布前影响预览 {#form.publish-preview}

> 列表行「发布」与表单「发布」均打开发布前影响预览弹窗。用户点击「确认发布」后写入数据。

**发布后将执行**（暂存不触发）：

| 动作 | 条件 | 数据变更 |
|------|------|----------|
| 生成新规则 | 始终 | 草稿 → 已发布；冻结价卡+费用项快照 |
| 修改旧规则失效时间 | 同客户+费用类型时间重叠 | 旧规则 `effective_end_time` 截断为「新 start 前一日 23:59:59」 |
| 不修改旧规则 | 无时间重叠 | — |

| 元素 | 说明 | 交互逻辑 |
|------|------|----------|
| 发布前影响预览 | 弹窗标题 | 展示发布后将执行、新规则、对已有规则的影响、规则快照 |
| 发布后将执行 | 摘要区 | 始终生成新规则；有重叠时同步修改旧规则失效时间 |
| 新规则 | 摘要区 + 状态徽章 | 配置名、客户、闭区间生效期、生效判定（开始日期 ≤ 今天→立即生效；> 今天→待生效） |
| 对已有规则的影响 | 表格或成功提示 | 无重叠：绿色提示不影响已有规则；有重叠：表格展示原失效时间→截断后时间 |
| 规则快照 | 说明区 | 发布时冻结价卡+费用项，生成不可变快照 |
| 取消 | 弹窗底部 | 关闭弹窗，不发布 |
| 确认发布 | 弹窗底部 | 执行：生成新规则 +（如有重叠）截断旧规则 + 冻结快照 |

## 4. Technical Specifications

- **Architecture Overview**：Flow 原型（list + form），Pattern 驱动渲染，Spec JSON 按类型拆分
- **Billing Match Rule**：`startTime ≤ event_time ≤ endTime`（闭区间）+ `customer` + `status=published`；结束默认 23:59:59，新规则 start 默认 +1 日 00:00:00
- **Integration Points**：原型阶段使用 mock 数据，localStorage 可选
- **Security & Privacy**：不涉及真实用户数据

## 5. Risks & Roadmap

### Phased Rollout

- **MVP**：列表 + 表单 + 详情弹窗 + 状态标签
- **v1.1**：发布冲突校验、作废复制新增
